# Roguelike 强化设计 · 波次 3 选 1

> 基于当前 Godot 数值：基础伤害约 `1～1.5`、射击 CD `0.38s`、怪血 `3 + floor((wave-1)/2)`、终极 CD 约 `20s`。  
> 目标：每波结束后停顿选强化，局内成长清晰，不破坏指尖即射手感。

---

## 1. 流程

```
清完本波目标 / 波次推进条件达成
  → 暂停刷怪与射击（或仅暂停刷怪）
  → 弹出 3 张强化卡（随机权重）
  → 体感悬停 1.2s / 键鼠点击 选定 1 张
  → 应用 → 下一波开始
```

### 抽卡规则（建议）

| 规则 | 建议 |
|------|------|
| 每波选项数 | 固定 **3** |
| 稀有度独立 roll | 每张卡单独判稀有度，再在该稀有池抽词条 |
| 同屏去重 | 同一 `upgrade_id` 不重复出现（不同稀有可算不同 id，或同 id 只出最高稀有） |
| 已叠满 | 达到硬上限的词条不再进入卡池 |
| 波次倾斜 | 波次越高，蓝/紫权重略升（见下表） |

### 稀有度权重（单卡）

| 波次 | 普通 Common | 稀有 Rare（蓝） | 史诗 Epic（紫） |
|------|-------------|-----------------|-----------------|
| 1–2 | 75% | 22% | 3% |
| 3–5 | 60% | 32% | 8% |
| 6–8 | 48% | 38% | 14% |
| 9+ | 38% | 42% | 20% |

颜色：普通灰白 / 稀有 `#4C9EFF` / 史诗 `#B57BFF`。

## 2. 玩家属性模型（建议实现）

```text
damage_mult          伤害倍率，默认 1.0
fire_rate_mult       攻速倍率，默认 1.0  → 实际 CD = base_cd / fire_rate_mult
ult_cd_mult          终极冷却倍率，默认 1.0 → 实际 ult_cd = base * ult_cd_mult
multishot_chance     连发触发概率 0~1
multishot_max        连发最大段数，硬顶 5（含首发则额外弹 0~4；见下）
```

### 射击公式

```text
base_cd      = 0.38
actual_cd    = max(0.12, base_cd / fire_rate_mult)     # 攻速软顶：约 0.12s
base_damage  = 1.0 + (0.5 if power>=0.75 else 0)
final_damage = base_damage * damage_mult

# 连发：每次「成功射出一发」后，以 multishot_chance 判定是否再射一发
# 可连续触发，最多再触发 (multishot_max - 1) 次额外弹
# 约定 multishot_max = 5 表示整段最多 5 发（1 主 + 4 额外）
extra = 0
while extra < multishot_max - 1 and rand() < multishot_chance:
    立刻再射一发（无前摇，可略微散射 ±6°）
    extra += 1
# 注意：连发段不重置基础射击 CD；整段共享同一次 CD
```

### 叠加上限（防爆炸）

| 属性 | 软提示 | 硬上限 |
|------|--------|--------|
| `damage_mult` | — | **3.0**（+200%） |
| `fire_rate_mult` | — | **2.5**（CD≈0.15，再被 0.12 截断） |
| `ult_cd_mult` | 越低越好 | **0.40**（终极最多 -60% CD） |
| `multishot_chance` | — | **0.75**（75%） |
| `multishot_max` | — | **5** |

| ID | 名称 | 稀有 | 效果 | 对 20s 终极 |
|----|------|------|------|-------------|

### 3.4 连发 `multishot`（概率连发，可连续触发，最多 5 发）

**规则（必须统一）：**

1. 每次**主动射击**（指尖/空格触发的那一下）射出第 1 发后开始判定。  
2. 以 `multishot_chance` 判定是否再射；成功则再判定，直到失败或达到 `multishot_max`。  
3. **整段最多 5 发**（1 主 + 最多 4 额外）。  
4. 额外弹：同一瞄准方向，可加 **±4°～8°** 微散射；伤害为 **主弹的 100%**（可用词条改成 70%/85% 做平衡）。  
5. 连发**不消耗**多次射击 CD，只占用一次 CD。  
6. 连发**不触发**「再开一段连发」（避免指数爆炸）；终极不走连发。

| ID | 名称 | 稀有 | 效果 |
|----|------|------|------|
| `ms_chance_c` | 回声火花 | 普通 | **连发概率 +8%** |
| `ms_chance_r` | 二重咏唱 | 稀有 | **连发概率 +14%** |
| `ms_chance_e` | 五重魔律 | 史诗 | **连发概率 +22%** |
| `ms_max_c` | 分叉咒 | 普通 | **连发上限 +1**（最多到 5） |
| `ms_max_r` | 分裂核心 | 稀有 | **连发上限 +1，且连发概率 +6%** |
| `ms_max_e` | 连珠圣典 | 史诗 | **连发上限 +2，且连发概率 +10%**（仍封顶 5） |

**默认起点：**

```text
multishot_chance = 0.0
multishot_max    = 1   # 只有主弹；拿到第一个 +上限 后变为 2
```

**期望额外弹数（几何分布，封顶 n-1 次成功）：**

在概率 `p`、最多 `k=multishot_max` 发时，期望发数：

\[
E = \sum_{i=0}^{k-1} p^{i} \quad (\text{主弹必出，再乘是否继续})
\]

更直观：主弹 1 + 期望连续成功次数（cap 在 k-1）。

| p | max=3 | max=5 |
|---|-------|-------|
| 20% | ≈1.24 发 | ≈1.25 发 |
| 40% | ≈1.56 发 | ≈1.64 发 |
| 60% | ≈1.96 发 | ≈2.30 发 |
| 75% | ≈2.31 发 | ≈3.05 发 |

设计意图：前期靠 **上限** 打开连发，中后期靠 **概率** 拉满手感；双紫也难稳定 5 连，避免清屏无脑。

## 7. 平衡校调原则

1. **每波期望战力 +12%～20%**（综合伤害×攻速×连发）。  
2. 怪血/波次密度应随波次略涨，避免 5 波后无脑。  
3. 连发是方差来源：上限词条略少、概率词条略多。  
4. 史诗不要同时给「巨额伤害 + 高攻速 + 高连发」一张卡；单卡专注一条主属性。  
5. 硬顶必须存在，防止无限叠乘。

### 快速自检 build

| Build | 关键卡 | 手感 |
|-------|------|------|
| 速射流 | 攻速 + 连发率 | 指尖机枪 |
| 重击流 | 伤害 + 穿透/爆炸 | 少而狠 |
| 大招流 | 终极 CD + 终极伤害 | 双手仪式频繁 |
| 异常流 | 灼烧/减速强化 | 火冰身份感 |

## 9. 一句话

**每波 3 选 1**：主轴四系为 **伤害 / 攻速 / 终极 CD / 连发（率+上限，连发最多 5）**；普通小步、稀有中步、史诗大步；再用生存、元素、弹道、终极协同做 build 分化。
